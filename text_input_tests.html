Text input tests

This page shows what features HTML Input element has and so that they must be preserved in kiji, and how they interfere or not with kiji's own shortcuts.

Goal is to keep inputs focused (e.g. Caption, or Left, or Font), while you selecting various items in canvas. This works because canvas is not focusable, but is in conflict with usefull shortcuts we want to use in canvas (e.g. Del, Arrows, ...) which must also works in input.

Note that you must test it here (as reference) and also in kiji (testing) and they must behave the same.

Typing any letters

You can type any UTF-8 letter into text input. Just press the key and it works. Also combined characters, like "č" or "Á" (press ´, then shift, then "a") or "9" or "{" or ">". You can also type any combination like \n, or &lt; and you see whatever you wrote, that is \n or &lt;

Shift + letter

While holding shift, you can type capital letters.

Left/right arrows

Using left or right arrow, you can move cursor inside input. Then you can insert characters where the cursor is.

Home and end keys

Pressing home set cursor to begining of input

CapsLock on + letters

While CapsLock is on, while you press letters they are inserted as capitals.

Ctrl + left/right arrows

While holding ctrl, you can move cursor to next/previous word inside input, but item itself does not move!

Shift + left/right arrows

While holding shift, you can select part of text inside input, but item itself does not move by 1px!

Moving selected element by 1px

While holding shift, you can move item by 1px by pressing arrow keys (left, right, up, down).

Ctrl + Shift + left/right arrows

While holding ctrl and shift, you should be able to select entire words by pressing left/right arrows.

Ctrl+a selects entire text

While holding ctrl, pressing "a" should select entire text in input, while not selecting all items.

Delete and backspace

Pressing delete or backspace while inside input delete text, not selected items.
